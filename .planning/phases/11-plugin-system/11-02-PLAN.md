  ```csharp
  internal sealed class RecrdPluginLoadContext : AssemblyLoadContext
  ```

  Constructor:
  - Takes `string pluginPath` (full path to the plugin's main DLL)
  - Calls `base(name: Path.GetFileNameWithoutExtension(pluginPath), isCollectible: true)` per D-07
  - Creates `_resolver = new AssemblyDependencyResolver(pluginPath)` — this reads the `.deps.json` in the same directory

  `Load(AssemblyName assemblyName)` override per D-07:
  - If `assemblyName.Name == "Recrd.Core"` return `null` — type unification, host's Recrd.Core is used
  - Otherwise call `_resolver.ResolveAssemblyToPath(assemblyName)` and if non-null, call `LoadFromAssemblyPath(path)`
  - Return `null` for anything the resolver cannot find (falls back to default context)

  This ensures:
  - Plugin types implementing `ITestCompiler` etc. share the host's type identity
  - Plugin-specific dependencies (e.g., ClosedXML) are isolated in the plugin's ALC
  - No DLL locking issues since we use ALC-based loading (not Assembly.LoadFile)
