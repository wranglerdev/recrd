  <name>Task 1: Add exception-safe compilation wrapper to PluginManager</name>
  <files>apps/recrd-cli/Plugins/PluginManager.cs</files>
  <action>
  Per D-11, add a method to PluginManager:

  ```csharp
  public async Task<CompilationResult> SafeCompileAsync(
      ITestCompiler compiler, Session session, CompilerOptions options)
  ```

  This method wraps `compiler.CompileAsync(session, options)` in a `try/catch(Exception ex)`:
  - **Success path:** Return the `CompilationResult` as-is
  - **Exception path:** Return a new `CompilationResult` with:
    - `GeneratedFiles`: empty list
    - `Warnings`: single item `$"[plugin:{compiler.TargetName}] {ex.Message}"`
    - `DependencyManifest`: empty dictionary

  This is the ONLY place plugin exceptions are caught. The method is called by CompileCommand when the selected compiler comes from a plugin (identified by checking if it's in the built-in set or not).

  Also add a helper method to identify plugin-provided compilers:
  ```csharp
  public bool IsPluginCompiler(ITestCompiler compiler)
  ```
  Returns true if the compiler was loaded from a plugin ALC (not a built-in). Implementation: check if the compiler's type is from an assembly loaded in a RecrdPluginLoadContext.

| Plugin CompileAsync -> Host process | Untrusted code execution; may throw any exception type |
| Plugin exception message -> Warning output | Exception message displayed to user; could contain misleading content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-11-02 | Denial of Service | Plugin CompileAsync | mitigate | try/catch(Exception) in SafeCompileAsync; caught exceptions become warnings, host continues with exit 0 |

| T-11-10 | Repudiation | Plugin exception message | accept | Exception messages logged as-is in warnings; no sanitization needed since output goes to the invoking user only |
