1. **Add `DataFilePath` to `GherkinGeneratorOptions`** (needed for GherkinException per D-06):
   - Add property: `public string? DataFilePath { get; init; }` to the existing `GherkinGeneratorOptions` record
   - This allows the caller to pass the data file path so the generator can include it in GherkinException messages

2. **Also add `TextWriter? WarningOutput` to `GherkinGeneratorOptions`** (per D-07 discretion):
   - Add property: `public TextWriter? WarningOutput { get; init; }` — when null, warnings go to `Console.Error`
   - This enables test injection of a StringWriter to capture stderr warnings

3. **Create `packages/Recrd.Gherkin/Internal/ExemplosTableBuilder.cs`**:
   - `namespace Recrd.Gherkin.Internal;`
   - `internal static class ExemplosTableBuilder`

   - Method: `internal static List<string> DeriveColumnOrder(IReadOnlyList<string> renderedStepTexts)`
     - Walk each rendered step text in order
     - Use `Regex.Matches(text, @"<([^>]+)>")` to find variable placeholders
     - Track seen variables with `HashSet<string>` and collect first-appearance order in `List<string>`
     - Return the ordered list — this is the Exemplos column order (GHER-09)

   - Method: `internal static string RenderHeader(IReadOnlyList<string> columnOrder)`
     - Return `"    | " + string.Join(" | ", columnOrder) + " |"` (4-space indent, pipe-delimited)

   - Method: `internal static string RenderRow(IReadOnlyList<string> columnOrder, IReadOnlyDictionary<string, string> row)`
     - For each column in `columnOrder`, look up `row[column]`; pad values to match header column widths
     - Return `"    | " + string.Join(" | ", values) + " |"` (4-space indent)
     - For column width alignment: compute max width per column across header + all rows. Pad each cell with spaces to match width. This ensures the table is visually aligned and deterministic.

   - Method: `internal static async Task<(List<string> ColumnOrder, List<IReadOnlyDictionary<string, string>> Rows)> MaterializeDataAsync(IDataProvider dataProvider, CancellationToken ct)`
     - Enumerate `dataProvider.StreamAsync(ct)` collecting all rows into a `List`
     - Column headers come from the first row's keys
     - Return `(columnOrder: first row keys as list, rows: all rows including first)`

4. **Extend `GherkinGenerator.GenerateAsync`** with the data-driven path:

   After writing steps (which already works from Plan 02), add the Exemplos section when `session.Variables.Count > 0`:

   a. **Validate data provider**: If `session.Variables.Count > 0` and `dataProvider is null`, throw `GherkinException` with message indicating no data provided for variables.

   b. **Materialize data**: Call `ExemplosTableBuilder.MaterializeDataAsync(dataProvider, ct)` to get column headers and rows.

   c. **Validate variable coverage (GHER-03)**: For each variable in `session.Variables`:
      - Check if `variable.Name` exists in the data column headers
      - If NOT found, throw `new GherkinException(variable.Name, options?.DataFilePath ?? "(unknown)", $"Variable '{variable.Name}' is declared in the session but missing from data columns. Available columns: {string.Join(", ", dataColumnHeaders)}")`

   d. **Warn on extra columns (GHER-04)**: Collect variable names from `session.Variables` into a HashSet. For each data column header NOT in that set:
      - Write to `options?.WarningOutput ?? Console.Error`: `$"Warning: data column '{column}' is not referenced by any variable in the session and will be ignored."`

   e. **Derive column order (GHER-09)**: Collect all rendered step texts (already computed for step emission). Call `ExemplosTableBuilder.DeriveColumnOrder(renderedStepTexts)` to get the first-appearance order.

   f. **Emit Exemplos table (GHER-02)**:
      ```
      await output.WriteLineAsync();
      await output.WriteLineAsync("    Exemplos:");
      ```
      - Compute column widths: for each column in `columnOrder`, max of column name length and max value length across all rows
      - Write header: `      | {col1 padded} | {col2 padded} |`
      - Write each row: `      | {val1 padded} | {val2 padded} |`
      - Use 6-space indent for Exemplos content (inside the Cenario block)

   g. **Important**: The step rendering must use `<variable_name>` placeholders in the body text (per D-02). When a step's Payload value matches a variable placeholder pattern `<varname>`, keep it verbatim in the step text. The StepTextRenderer already handles this — values from Payload are emitted as-is.
