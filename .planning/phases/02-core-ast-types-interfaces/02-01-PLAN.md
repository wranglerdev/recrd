**SessionSerializationTests.cs** (CORE-01):
- `using Recrd.Core.Ast;` and `using Recrd.Core.Serialization;`
- Test: `Session_RoundTrips_WithAllFields` -- create a Session with schemaVersion=1, metadata (id, createdAt, browserEngine, viewportSize, baseUrl), empty variables list, and a list with one ActionStep. Serialize via `JsonSerializer.Serialize(session, RecrdJsonContext.Default.Session)`, deserialize back, assert all fields match including SchemaVersion == 1.
- Test: `Session_RoundTrips_WithPolymorphicSteps` -- create Session with one ActionStep, one AssertionStep, one GroupStep in steps list. Round-trip serialize/deserialize. Assert deserialized steps are correct types via `Assert.IsType<ActionStep>`, `Assert.IsType<AssertionStep>`, `Assert.IsType<GroupStep>`.
- Test: `Session_RoundTrips_WithVariables` -- Session with 2 Variables. Assert variables round-trip with correct names.
- Test: `Session_Serialization_EmitsCamelCaseKeys` -- serialize a Session, assert raw JSON string contains `"schemaVersion"` (camelCase), `"metadata"`, `"steps"`, `"$type"`.

**StepModelTests.cs** (CORE-02, CORE-03, CORE-04):

- Static property: `public static TheoryData<ActionType> AllActionTypes => new(Enum.GetValues<ActionType>());`
- Theory test: `ActionStep_IsConstructible_ForAllSubtypes(ActionType actionType)` -- construct `new ActionStep(actionType, testSelector, new Dictionary<string, string>())`, assert `step.ActionType == actionType`. Per D-02, testSelector is a Selector with SelectorStrategy.DataTestId.
- Static property: `public static TheoryData<AssertionType> AllAssertionTypes => new(Enum.GetValues<AssertionType>());`
- Theory test: `AssertionStep_IsConstructible_ForAllSubtypes(AssertionType assertionType)` -- construct, assert type matches. Per D-03.
- Fact test: `GroupStep_ContainsChildSteps` -- create GroupStep with GroupType.Given and a list containing one ActionStep. Assert `groupStep.Steps.Count == 1` and `groupStep.GroupType == GroupType.Given`. Per D-04.
- Theory test: `GroupStep_IsConstructible_ForAllGroupTypes` with `TheoryData<GroupType>` from `Enum.GetValues<GroupType>()`.
- Fact test: `ActionType_HasSixValues` -- `Assert.Equal(6, Enum.GetValues<ActionType>().Length)` (Click, Type, Select, Navigate, Upload, DragDrop per CORE-02).
- Fact test: `AssertionType_HasFiveValues` -- `Assert.Equal(5, Enum.GetValues<AssertionType>().Length)` (TextEquals, TextContains, Visible, Enabled, UrlMatches per CORE-03).

**SelectorVariableTests.cs** (CORE-05, CORE-06):
- `using Recrd.Core.Ast;`
- Fact test: `Selector_PriorityArray_RanksCorrectly` -- create a Selector with strategies `[DataTestId, Id, Role, Css, XPath]`, assert the first strategy is `SelectorStrategy.DataTestId` and last is `SelectorStrategy.XPath`. Per CORE-05.
- Fact test: `Variable_ValidName_Succeeds` -- `new Variable("my_var_1")` does not throw. Per CORE-06.
- Theory test: `Variable_InvalidName_ThrowsArgumentException` with inline data: `""`, `"A"`, `"1abc"`, `"my-var"`, `"a" + new string('b', 64)` (65 chars total, exceeds 64 max). Each should throw `ArgumentException`. Per CORE-06 regex `^[a-z][a-z0-9_]{0,63}$`.
- Fact test: `Variable_NameRegex_AcceptsBoundary` -- `new Variable("a")` (1 char) and `new Variable("a" + new string('b', 63))` (64 chars) both succeed.
- Fact test: `SelectorStrategy_HasFiveValues` -- `Assert.Equal(5, Enum.GetValues<SelectorStrategy>().Length)` (DataTestId, Id, Role, Css, XPath per CORE-05).

**ChannelPipelineTests.cs** (CORE-11, CORE-12):
- `using Recrd.Core.Pipeline;`
- Fact test: `RecordingChannel_WriteAndRead_SingleEvent` -- create `new RecordingChannel(capacity: 2)`, write one RecordedEvent, call Complete(), ReadAllAsync into list, assert single item with correct Id. Per CORE-11.
- Fact test: `RecordingChannel_Backpressure_BlocksWhenFull` -- create channel with capacity=1, write one event (fills it), start a second WriteAsync (should not complete immediately), verify via `Task.WhenAny` with a short delay that it's still pending, then read one to unblock, complete. Per CORE-11 backpressure.
- Fact test: `RecordingChannel_Cancellation_StopsRead` -- create channel, start ReadAllAsync with a CancellationTokenSource, cancel after short delay, assert OperationCanceledException is thrown. Per CORE-11 cancellation.
- Fact test: `RecordingChannel_DrainWithoutDeadlock` -- write 5 events, Complete(), read all with 5-second timeout CancellationToken, assert 5 events collected. Per CORE-11.
- Fact test: `RecordedEvent_HasAllRequiredFields` -- construct a RecordedEvent with Id (Guid string), TimestampMs (long), EventType (enum), Selectors (list of Selector), Payload (dictionary), DataVariable (nullable string). Assert all properties accessible. Per CORE-12.
