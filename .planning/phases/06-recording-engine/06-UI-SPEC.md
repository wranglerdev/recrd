# Phase 6 — UI Design Contract

> Visual and interaction contract for the recording engine inspector side-panel and in-page overlay UI.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope

Phase 6 has exactly two UI surfaces, both implemented as vanilla HTML/CSS/JS (no build step, no npm, no framework):

| Surface | Delivery | Requirement |
|---------|----------|-------------|
| Inspector side-panel | Single self-contained `.html` embedded resource; served to secondary BrowserContext via `Page.RouteAsync` | REC-11, REC-12, REC-13, REC-14 (source: CONTEXT.md D-06) |
| In-page overlay menus | Injected by the JS recording agent into the recording page; covers right-click "Tag as Variable" and "Add Assertion" in pause mode | REC-13, REC-14 (source: CONTEXT.md D-08, D-09) |

All other Phase 6 deliverables (`IRecorderEngine`, channel pipeline, selector extraction, snapshot timer) are pure C# with no UI surface.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-adjacent gaps, inline badge padding |
| sm | 8px | Row padding, compact list items |
| md | 16px | Panel body padding, form field spacing |
| lg | 24px | Section divider gaps |
| xl | 32px | Panel header height reference |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Right-click overlay menu items: minimum touch/click target height 32px (sm + sm padding), to guarantee pointer accuracy in browser context.
- Overlay menu total max-height: 200px with `overflow-y: auto` to handle long variable name lists.

## Color

Dark theme is the sole color scheme for both surfaces. Rationale: the inspector panel opens alongside arbitrary websites; a dark panel creates a clear visual boundary between recorded site and tooling chrome. The recording-page overlay uses semi-transparent dark to float above any page background.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` | Inspector panel background, overlay menu background |
| Secondary (30%) | `#2a2a2a` | Event stream row background (alternating), variable list item background |
| Accent (10%) | `#3b82f6` (blue-500) | Recording indicator dot, "Tag as Variable" confirm button, active variable badge border |
| Destructive | `#ef4444` (red-500) | Duplicate variable name warning banner, "stop recording" visual state |

**Accent reserved for:** recording-active state indicator, the "Tag as Variable" primary confirm action, and the border on variable badge chips. Must not be used for general text, list items, or section backgrounds.

**Text on dominant background:** `#e5e5e5` (near-white) for body text; `#a3a3a3` (neutral-400) for secondary/dimmed text (e.g., timestamps, selector strategy names).

**Border color:** `#3a3a3a` for dividers and panel borders. No box-shadows — borders only.

## Component Inventory

### Inspector Side-Panel

The inspector panel is a single `<html>` page served as an embedded resource. It must be fully functional with no external resource loads (no CDN, no Google Fonts, no external scripts).

**Primary focal point:** The recording state badge in the panel header is the primary visual anchor when the inspector is in idle state. It is the first element a developer reads to determine session status.

| Component | Element | States |
|-----------|---------|--------|
| Panel header | `<header>` with title "Inspector" + recording state badge | recording (accent dot + "REC"), paused ("PAUSE"), idle ("IDLE") |
| Event stream list | `<ul>` with `<li>` rows; each row shows: event type, element selector (truncated at 40 chars), timestamp delta | normal, hover (secondary bg), selected |
| Variable list | `<section>` below event stream; shows all named variables as chips | default, duplicate-warning (destructive border) |
| "Tag as Variable" form | `<form>` inside a `<dialog>` overlay; single `<input>` for name, two buttons | default, invalid (name violates `^[a-z][a-z0-9_]{0,63}$`), duplicate-error |
| Assertion builder | `<form>` inside a `<dialog>` overlay; assertion type `<select>` + selector display + expected value `<input>` | pause-mode-only; hidden in recording mode |

### In-Page Right-Click Overlay (recording page)

Injected by the JS agent. Must not interfere with existing page styles. Use a scoped CSS class prefix `__recrd__` on all injected elements to prevent collisions.

| Component | Element | States |
|-----------|---------|--------|
| Context menu | `<div class="__recrd__menu">` positioned absolute at cursor | visible (recording mode), visible (pause mode — adds "Add Assertion" item), hidden |
| "Tag as Variable" menu item | `<button>` | default, hover |
| "Add Assertion" menu item | `<button>` — only rendered in pause mode | default, hover |
| Duplicate name warning | `<div class="__recrd__warning">` positioned below input | hidden, visible (destructive bg, white text) |

## Interaction Contracts

### Inspector Panel Update Model

C# calls `Page.EvaluateAsync("window.__recrdPush({json})")` on the inspector page each time a `RecordedEvent` arrives on the channel (source: CONTEXT.md D-07).

- The inspector page must define `window.__recrdPush(event)` as a global function.
- The event stream list scrolls to the newest item automatically (`scrollIntoView({ behavior: 'smooth' })`).
- Maximum 500 events rendered in the DOM at one time; older rows removed from the top of the list (no pagination UI needed — events are a live tail).

### Variable Tag Flow

1. User right-clicks a field in the recording page.
2. JS agent intercepts `contextmenu`, suppresses default browser menu, renders `__recrd__menu` overlay at cursor position.
3. User clicks "Tag as Variable".
4. JS agent fires `window.__recrdCapture({ type: 'tag-start', selector: {...}, value: '...' })`.
5. C# receives tag-start event, calls `Page.EvaluateAsync("window.__recrdShowTagDialog({json})")` on the inspector page.
6. Inspector renders the "Tag as Variable" dialog.
7. User types a name and clicks "Tag".
8. Inspector calls `window.__recrdCapture({ type: 'tag-confirm', name: '...' })` via `Page.ExposeFunctionAsync` (source: CONTEXT.md D-04).
9. C# validates uniqueness; if duplicate, calls `Page.EvaluateAsync("window.__recrdTagError('duplicate')")` — inspector shows warning without closing dialog.
10. On success: C# replaces literal with variable placeholder in AST; inspector closes dialog and adds variable chip.

### Pause Mode Assertion Flow

1. User issues `recrd pause` (CLI signal or future keyboard shortcut — out of scope for this phase's UI).
2. C# calls `Page.EvaluateAsync("window.__recrdSetMode('pause')")` on the recording page.
3. JS agent updates internal mode flag; subsequent right-click menus include "Add Assertion" item.
4. User right-clicks, selects "Add Assertion".
5. JS agent fires `window.__recrdCapture({ type: 'assert-start', selector: {...}, textContent: '...' })`.
6. C# calls `Page.EvaluateAsync("window.__recrdShowAssertDialog({json})")` on the inspector page.
7. User selects assertion type and expected value, clicks "Add Assertion".
8. Inspector calls `window.__recrdCapture({ type: 'assert-confirm', assertionType: '...', expected: '...' })`.
9. C# inserts `AssertionStep` into AST; inspector closes dialog and appends event to stream.

## Accessibility

These surfaces run inside a Playwright-controlled BrowserContext used exclusively by the developer operating the tool — not by end users. WCAG AA compliance is not required. However, the following baseline applies:

- All interactive elements (`<button>`, `<input>`, `<select>`) are keyboard-reachable within each surface.
- Error and warning messages are rendered as visible text (not only color).
- The right-click overlay menu is dismissed on `Escape` keypress.
