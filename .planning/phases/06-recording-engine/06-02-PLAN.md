Create `packages/Recrd.Recording/Scripts/recording-agent.js` as an EmbeddedResource file. This script is injected via `BrowserContext.AddInitScriptAsync` and runs in every frame.

The agent must:

1. **Define global state:**
   - `window.__recrdMode = 'record'` (set externally via `Page.EvaluateAsync("window.__recrdSetMode('pause')")`)
   - Guard: if `window.__recrdCapture` is not defined (function not yet exposed), silently skip event capture

2. **Selector extraction function** `__extractSelectors(element)`:
   Returns a JSON object `{ strategies: string[], values: { [strategy]: string } }` where strategies are ordered by priority:
   - `DataTestId`: `element.getAttribute('data-testid')` — if present, add `"DataTestId"` to strategies, value is the attribute value
   - `Id`: `element.id` — if non-empty, add `"Id"`, value is `#${element.id}`
   - `Role`: `element.getAttribute('role')` or inferred from tag (`button` -> `button`, `a` -> `link`, `input` -> `textbox`/`checkbox`/`radio` based on type) — if determinable, add `"Role"`, value is the role string
   - `Css`: Build CSS selector from tag + classes: `tagName.class1.class2` (max 3 classes) — always available, add `"Css"`
   - `XPath`: Build absolute XPath from element position: walk up to body, count sibling position per tag — always available, add `"XPath"`

   Priority order in the `strategies` array: DataTestId first (if present), then Id, Role, Css, XPath. Minimum 3 strategies guaranteed since Css and XPath are always extractable plus at least one of DataTestId/Id/Role/tag-inferred.

3. **7 event listeners** registered on `document` in capture phase (`addEventListener(type, handler, true)`):
   - `click`: Capture target element, extract selectors, send `{ type: 'Click', selectors: ..., payload: {} }`
   - `input` + `change`: Capture on input/textarea/select elements; send `{ type: 'InputChange', selectors: ..., payload: { value: element.value } }`
   - `change` on `<select>`: Send `{ type: 'Select', selectors: ..., payload: { value: element.value, text: element.options[element.selectedIndex].text } }`
   - `mouseover` with explicit-only filter (only fire if element has `data-recrd-hover="true"` attribute): Send `{ type: 'Hover', selectors: ..., payload: {} }`. Note: hover is opt-in to avoid noise.
   - Navigation: Listen to `beforeunload` for page navigations. Send `{ type: 'Navigation', selectors: [], payload: { url: location.href } }`. Also observe `popstate` for SPA nav.
   - `change` on `<input type="file">`: Send `{ type: 'FileUpload', selectors: ..., payload: { files: Array.from(element.files).map(f => f.name).join(',') } }`
   - `dragend`: Send `{ type: 'DragDrop', selectors: ...(source), payload: { targetSelector: __extractSelectors(event.target) } }`

4. **Event dispatch function** — for each captured event:
   - Check `window.__recrdMode === 'record'` — if not, skip (except for tag/assert events which have their own type)
   - Generate a unique ID: `'evt-' + Date.now() + '-' + Math.random().toString(36).slice(2, 8)`
   - Call `window.__recrdCapture(JSON.stringify({ id, timestamp: performance.now(), type, selectors, payload }))`

5. **Right-click overlay menu** (per D-08, D-09):
   - Listen to `contextmenu` on document in capture phase
   - If `window.__recrdMode === 'record'` or `'pause'`: `event.preventDefault()`
   - Render a `<div class="__recrd__menu">` positioned at `event.clientX, event.clientY`
   - Always show "Tag as Variable" button
   - If `window.__recrdMode === 'pause'`: also show "Add Assertion" button
   - "Tag as Variable" click: call `window.__recrdCapture(JSON.stringify({ id: ..., timestamp: ..., type: 'TagStart', selectors: __extractSelectors(targetElement), payload: { value: targetElement.value || targetElement.textContent } }))`
   - "Add Assertion" click: call `window.__recrdCapture(JSON.stringify({ id: ..., timestamp: ..., type: 'AssertStart', selectors: __extractSelectors(targetElement), payload: { textContent: targetElement.textContent } }))`
   - Dismiss menu on `Escape` key or click outside

6. **Mode control functions** (called from C# via `Page.EvaluateAsync`):

   - `window.__recrdShowTagDialog` and `window.__recrdShowAssertDialog` — these are called from C# on the inspector page (not the recording page). On the recording page, tag/assert starts are initiated by the context menu.

7. **CSS for overlay menu** — inject a `<style>` element with `__recrd__` prefixed classes:

using Recrd.Core.Ast;
using Recrd.Core.Pipeline;

namespace Recrd.Recording.Engine;

internal class SessionBuilder
{
    private readonly List<IStep> _steps = new();
    private readonly List<Variable> _variables = new();
    private readonly SessionMetadata _metadata;

    public SessionBuilder(SessionMetadata metadata) { _metadata = metadata; }

    public void AddStep(IStep step) => _steps.Add(step);
    public void AddVariable(Variable variable) => _variables.Add(variable);
    public bool HasVariable(string name) => _variables.Any(v => v.Name == name);

    public IStep ConvertToStep(RecordedEvent evt)
    {
        // Map RecordedEventType to ActionStep:
        //   Click -> ActionStep(ActionType.Click, evt.Selectors[0], evt.Payload)
        //   InputChange -> ActionStep(ActionType.Type, ...)
        //   Select -> ActionStep(ActionType.Select, ...)
        //   Hover -> ActionStep(ActionType.Click, ...) // hover is recorded as-is
        //   Navigation -> ActionStep(ActionType.Navigate, ...)
        //   FileUpload -> ActionStep(ActionType.Upload, ...)
        //   DragDrop -> ActionStep(ActionType.DragDrop, ...)
        // Uses evt.Selectors[0] as the primary Selector (first in ranked list)
    }

    public Session Build() => new Session(
        SchemaVersion: 1,
        Metadata: _metadata,
        Variables: _variables.AsReadOnly(),
        Steps: _steps.AsReadOnly());
}
