    **Update `apps/recrd-cli/Commands/ValidateCommand.cs`** (per CLI-04):

    Replace the stub SetAction with:

    1. Parse `<session>` argument (FileInfo). Validate file exists. Canonicalize with `Path.GetFullPath`. Validate extension is `.recrd`.
    2. Read file content: `await File.ReadAllTextAsync(sessionPath, ct)`
    3. Attempt deserialization:
       ```csharp
       Session? session;
       try
       {
           session = JsonSerializer.Deserialize(json, RecrdJsonContext.Default.Session);
       }
       catch (JsonException ex)
       {
           CliOutput.WriteError($"Invalid JSON: {ex.Message}");
           return 1;
       }
       ```
    4. If `session is null`: `CliOutput.WriteError("Session file is empty or invalid.")`, return 1
    5. **Schema validation:**
       - `session.SchemaVersion` must be 1: if not, `CliOutput.WriteError($"Unsupported schema version: {session.SchemaVersion}. Expected: 1")`, return 1
       - `session.Metadata` must not be null
       - `session.Steps` must not be empty (at least 1 step)
    6. **Variable consistency validation:**
       - For each Variable in `session.Variables`, validate the name matches `^[a-z][a-z0-9_]{0,63}$` (the Variable constructor already enforces this, but deserialization might bypass it — re-check with regex)
       - Check that each Variable.StepRef (if non-null) refers to an existing step index or is valid
    7. On all checks passing: `CliOutput.WriteSuccess("Session is valid.")`, return 0
    8. On any failure: print specific error message via `CliOutput.WriteError` naming the issue, return 1

    1. Parse `<session>` argument (FileInfo) and optional `--out` (FileInfo?). Validate file exists. Canonicalize.

       var json = await File.ReadAllTextAsync(sessionPath, ct);

       - For `ActionStep`: create new ActionStep with same ActionType, sanitized Selector (see below), and empty Payload (`new Dictionary<string, string>()`)
       - For `AssertionStep`: create new AssertionStep with same AssertionType, sanitized Selector, and empty Payload (`new Dictionary<string, string>()`)
       - For `GroupStep`: create new GroupStep with same GroupType and recursively sanitized child Steps
       - **Sanitize Selector:** create new Selector with same Strategies list but Values dictionary where every value is replaced with `"***"`: `new Dictionary<SelectorStrategy, string>(selector.Values.Select(kv => KeyValuePair.Create(kv.Key, "***")))`
       - **Keep Variables unchanged** (they contain names and step refs, not literal values)
       - **Keep Metadata unchanged** (baseUrl, browserEngine, etc. are config, not PII)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI file arg -> file system | User-supplied .recrd path, --data path, --out path enter file system operations |
| Session file content -> deserialization | Untrusted JSON content deserialized into typed Session objects |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-06 | Tampering | Session file path argument | mitigate | Canonicalize with Path.GetFullPath; validate .recrd extension before processing |
| T-8-07 | Tampering | Malformed session JSON | mitigate | Wrap deserialization in try/catch JsonException; report specific error, exit non-zero |
| T-8-08 | Information Disclosure | sanitize output path traversal | mitigate | Canonicalize --out path with Path.GetFullPath; output directory must exist |
| T-8-09 | Information Disclosure | Incomplete sanitization | mitigate | Sanitize both Selector.Values AND step Payload dictionaries; replace values with "***" not empty string (empty string could be confused with legitimate empty value) |
</threat_model>
