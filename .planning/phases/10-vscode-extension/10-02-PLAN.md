<task type="auto">
  <name>Task 1: Implement CliRunner wrapper</name>
  <files>apps/vscode-extension/src/cli/runner.ts</files>
  <action>
    Create a robust wrapper for child_process.spawn.
    - Implement CliRunner class with a static OutputChannel 'recrd'.
    - Implement spawn(args: string[]) that returns a ChildProcess but also manages output streaming.
    - Ensure it handles stdout/stderr by appending to the OutputChannel.
    - Provide a way to wait for exit or manually kill the process.
    - Path to 'recrd' should be resolved from 'recrd.executablePath' config.
    - Per VSCE-06: Strict CLI-only communication via stdout/stderr.
  </action>
  <verify>
    <automated>cd apps/vscode-extension && node esbuild.js && grep -r "child_process" dist/</automated>
  </verify>
  <done>CliRunner can spawn processes and stream output to a VS Code OutputChannel.</done>
</task>

<task type="auto">
  <name>Task 2: Implement Start and Stop Commands</name>
  <files>apps/vscode-extension/src/commands/recording.ts, apps/vscode-extension/src/extension.ts</files>
  <action>
    Implement recording lifecycle commands.
    - Create 'recrd.startRecording' command that:
        - Prompts for Base URL if not provided (optional for now).
        - Spawns 'recrd start' with appropriate flags.
        - Updates StatusBarManager to 'Recording' state.
        - Stores the process reference to allow stopping.
    - Create 'recrd.stopRecording' command that:
        - Sends signal to stop (if CLI supports) or kills the process.
        - Actually 'recrd stop' should be called if 'recrd start' is long-running, or however the CLI was designed in Phase 8.
        - Updates StatusBarManager to 'Idle' state.
    - Register commands in extension.ts and package.json.
  </action>
  <verify>
    <automated>cd apps/vscode-extension && node esbuild.js && grep -r "recrd.startRecording" dist/</automated>
  </verify>
  <done>Recording can be started and stopped via VS Code commands.</done>
</task>
