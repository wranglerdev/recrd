Replace stub in `packages/Recrd.Compilers/RobotSeleniumCompiler.cs` with full implementation. Follow the same pattern as `RobotBrowserCompiler` but with SeleniumLibrary-specific differences.

`CompileAsync(Session session, CompilerOptions options)`:
1. `Directory.CreateDirectory(options.OutputDirectory)`
2. Resource path: `Path.Combine(outDir, "session.resource")`
3. Robot path: `Path.Combine(outDir, "session.robot")`
4. `List<string> warnings`, `HashSet<string> emittedKeywords` for collision detection

**Write .resource file** (using `new StreamWriter(path, false, new UTF8Encoding(false))`):
```
{HeaderEmitter.Emit("robot-selenium", options.SourceFilePath, session)}
*** Settings ***
Library    SeleniumLibrary

*** Keywords ***
Abrir Suite
    Open Browser    ${BASE_URL}    chrome
    Set Selenium Implicit Wait    ${TIMEOUT}s

Fechar Suite
    Close All Browsers
```

Key difference from Browser compiler:
- `Library    SeleniumLibrary` (not `Browser`)
- `Open Browser    ${BASE_URL}    chrome` (not `New Browser`/`New Page`)
- `Set Selenium Implicit Wait    ${TIMEOUT}s` in Abrir Suite keyword (per D-08)
- `Close All Browsers` (not `Close Browser`)

Then for each step in `FlattenSteps(session.Steps)`:
- ActionStep: keyword name via `KeywordNameBuilder.Build`, body via `SeleniumKeywordEmitter.WriteKeywordAsync`
- AssertionStep: keyword name via `KeywordNameBuilder.BuildAssertion`, body via `SeleniumKeywordEmitter.WriteAssertionKeywordAsync`

Handle keyword name collisions with numeric suffix (same as Browser compiler).

**Write .robot file** — identical structure to Browser compiler except:
- Header target name: `"robot-selenium"`
- Same Settings/Variables/Test Cases structure

Return `new CompilationResult(generatedFiles: [resourcePath, robotPath], warnings: warnings, dependencyManifest: new Dictionary<string,string> { ["robotframework"] = "7.x", ["robotframework-seleniumlibrary"] = "6.x" })`.

Use `FlattenSteps` — either share from a common base utility or duplicate the static helper. Given both compilers are in the same assembly, a shared `internal static` utility in a `StepFlattener.cs` or directly in each compiler is fine.
