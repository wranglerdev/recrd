<task type="auto">
  <name>Task 1: Create release.yml workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
    Create .github/workflows/release.yml that triggers on tag pushes matching 'v*'.
    It should define a matrix for:
    - { os: windows-latest, rid: win-x64, pkg_type: zip }
    - { os: macos-latest, rid: osx-x64, pkg_type: zip }
    - { os: macos-latest, rid: osx-arm64, pkg_type: zip }
    - { os: ubuntu-latest, rid: linux-x64, pkg_type: tar.gz }
    The job must:
    1. Checkout the repo
    2. Setup .NET 10
    3. Run `dotnet publish apps/recrd-cli/recrd-cli.csproj -c Release -r <RID> --self-contained -p:PublishSingleFile=true -o ./publish/<RID>`
    4. Ensure .playwright/ directory is present in output (may require a separate `playwright install` or `dotnet build` step in some platforms, check if Microsoft.Playwright handles RID-specific driver download correctly)
    5. Zip/Tar the contents of ./publish/<RID>
  </action>
  <verify>
    <automated>test -f .github/workflows/release.yml</automated>
  </verify>
  <done>Release workflow file created with RID matrix and publish steps</done>
</task>

<task type="auto">
  <name>Task 2: Implement RID-specific packaging</name>
  <files>.github/workflows/release.yml</files>
  <action>
    Add a step in the release workflow to include the .playwright directory in the package.
    For each platform, make sure the .playwright directory exists in the output.
    Note: On Linux/macOS runners, .playwright is usually in the same directory as the binary after a build.
    If PublishSingleFile=true is used, the .playwright/ must be outside the binary.
    Run `ls -R ./publish/<RID>` before zipping to verify.
    The final zip/tar.gz file should have a name like `recrd-<version>-<RID>.<ext>`.
  </action>
  <verify>
    <automated>grep -q "dotnet publish" .github/workflows/release.yml && grep -q ".playwright" .github/workflows/release.yml</automated>
  </verify>
  <done>Packaging step includes .playwright folder and handles RID naming</done>
</task>
