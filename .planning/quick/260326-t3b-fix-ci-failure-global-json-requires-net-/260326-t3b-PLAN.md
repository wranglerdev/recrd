  <name>Task 1: Update global.json rollForward policy to latestFeature</name>
  <files>global.json</files>
  <action>
    Update global.json to use `"rollForward": "latestFeature"` instead of `"latestPatch"`.
    This allows the SDK resolver to accept any .NET 10.0.x feature band (100, 200, 300, etc.)
    as long as the major.minor matches.

    Keep the version pinned at `10.0.100` (base of the 100-band) as the minimum floor.
    With `latestFeature`, the resolver will use the latest available SDK >= 10.0.100 within
    the 10.0 major.minor range.

    The resulting global.json should be:
    ```json
    {
      "sdk": {
        "version": "10.0.100",
        "rollForward": "latestFeature"
      }
    }
    ```

    Why `latestFeature` over `latestPatch`: `latestPatch` constrains to the same feature band
    (e.g., 10.0.1xx). GitHub Actions `setup-dotnet` with `10.0.x` installs whatever is latest,
    which may be in a different feature band (10.0.2xx). `latestFeature` accepts any feature band
    within 10.0, which matches the CI `10.0.x` wildcard intent.

    Why version `10.0.100` not `10.0.103`: With `latestFeature`, the version is a minimum floor.
    Using the base of the 100-band (10.0.100) is the cleanest minimum that any .NET 10 SDK satisfies.
  </action>
