<objective>
Fix the CI coverage gate so it actually enforces the 90% line coverage threshold.

**Root cause:** The four coverage gate steps in `ci.yml` pass `--threshold 90 --threshold-type line --threshold-stat minimum` as flags to `dotnet test`. These flags are `coverlet.console`-only flags — they are NOT recognised by `dotnet test` or `coverlet.collector`. The SDK rejects them with `MSB1001: Unknown option` and aborts the step before any tests run, making the gate a no-op on CI. (Locally the `MSBUILD : error MSB1001` is visible.)

**Fix:** Switch the four gated test projects from `coverlet.collector` to `coverlet.msbuild`. The MSBuild integration honours `/p:Threshold`, `/p:ThresholdType`, `/p:ThresholdStat` properties passed to `dotnet test` and exits non-zero on breach. Update `ci.yml` coverage gate steps accordingly.

**Threshold note:** Current line coverage is below 90% on all four projects (implementations are stubs for phases not yet delivered). Keep the 90% target — the gate will correctly fail on stub code and will be green once implementation phases are complete. This is the intended behaviour: CI acts as a forcing function.

Purpose: Coverage gate must actually enforce thresholds, not silently pass everything.
Output: Updated .csproj files + ci.yml; `dotnet test /p:Threshold=90 ...` exits 1 on breach.
</objective>

The four coverage gate steps currently use unsupported flags that cause MSB1001 errors. Replace each step's `run:` block with the correct coverlet.msbuild property syntax.

Replace EACH of the four coverage gate steps. Current broken pattern:
```yaml
run: >
  dotnet test tests/Recrd.XYZ.Tests/Recrd.XYZ.Tests.csproj
  --no-build
  --collect:"XPlat Code Coverage"
  --threshold 90
  --threshold-type line
  --threshold-stat minimum
```

Replace with (the `--collect` flag is removed — coverlet.msbuild does not use the data collector):
```yaml

  dotnet test tests/Recrd.XYZ.Tests/Recrd.XYZ.Tests.csproj

  /p:CollectCoverage=true
  /p:CoverletOutputFormat=cobertura
  /p:Threshold=90
  /p:ThresholdType=line
  /p:ThresholdStat=minimum
```

Apply this pattern to all four gates: Recrd.Core, Recrd.Data, Recrd.Gherkin, Recrd.Compilers.
